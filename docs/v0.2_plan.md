# NexaFlow v0.2 开发计划

## 当前已落地

- 结构化会话聚合：新增 `flow_sessions_5s`，默认写入窗口 Top 500 五元组，支持通过采集器运行时配置 `session_topn` 调整到 20-5000，包含源/目的、端口、协议、服务、风险、方向、客户端、服务端和样例会话。
- 会话查询升级：`/api/v1/traffic/sessions` 优先读取 `flow_sessions_5s`，旧 `dimension_traffic_5s` flow 维度作为回退。
- 应用分析升级：应用分析、公网访问、事件上下文等依赖会话查询的功能开始使用结构化会话字段。
- 采集质量增强：Collector 在 Linux 环境读取 `/sys/class/net/<iface>/statistics` 的 RX/TX bytes、packets、dropped、errors，按窗口写入 `capture_quality_5s`。
- 采集质量展示：新增 `/api/v1/system/capture-quality`，并在数据质量页面展示接口 RX/TX 流量、丢包、错误、状态和建议。
- 操作审计底座：新增 `operation_audit` 表和 `/api/v1/system/audit-events`，记录采集配置、资产元数据、事件处置、规则、告警阈值和白名单/静默名单变更，前端新增“审计日志”页面。
- 登录保护底座：新增可选 `NEXAFLOW_AUTH_PASSWORD` 登录开关、HttpOnly 会话 Cookie、登录/退出接口和前端登录页；默认关闭，设置密码后保护 `/api/v1/*`。
- 测试增强：增加聚合器默认/运行时 TopFlow 保留量测试和服务分析会话聚合测试。

## 继续深化项

- 保存更多或全量会话：当前已支持可配置 TopN 会话追踪，但还不是全量流表，后续可扩展轻量全量流索引或采样策略。
- 更细采集统计：当前已覆盖 Linux 网卡接口统计，后续需要继续拆分 socket 队列丢包、BPF 过滤前后数量、用户态队列积压。
- 配置持久化：规则、白名单、采集器配置、资产元数据仍需逐步收敛到 PostgreSQL 或统一配置存储。
- 权限审计：操作审计日志和可选登录保护已落地，后续继续补角色权限、用户管理和更细的授权边界。
- 前端性能：当前为单页大包，后续应按视图拆分动态加载，降低首次加载体积。

## 建议优先级

1. 会话表继续扩展到近实时全量轻量聚合或可控采样策略。
2. 采集质量指标拆分并展示驱动、socket、过滤、聚合队列四层状态。
3. 把事件中心和资产风险的推荐动作接入结构化会话上下文。
4. API 和存储层补更系统的单元测试。
5. 前端路由拆分和权限审计。
